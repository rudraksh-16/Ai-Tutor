LOGISTIC REGRESSION A COMPLETE BOOK-STYLE GUIDE

  --------------------------------------------------
  CHAPTER 1: INTRODUCTION TO LOGISTIC REGRESSION
  --------------------------------------------------
  Logistic Regression is one of the most fundamental
  algorithms in machine learning and statistics. It
  is a supervised learning algorithm primarily used
  for classification problems where the target
  variable is categorical, most commonly binary.

  Unlike linear regression, which predicts
  continuous values, logistic regression predicts
  the probability that an observation belongs to a
  particular class. These probabilities are then
  mapped to class labels using a decision threshold.

  Real-world examples include: - Determining whether
  an email is spam or not - Predicting whether a
  patient has a disease - Deciding whether a
  customer will churn
  --------------------------------------------------

CHAPTER 2: LINEAR MODEL FOUNDATION

At its core, logistic regression is based on a linear model. Given a set
of input features, the algorithm computes a weighted sum of the inputs.

Mathematically: z = w1x1 + w2x2 + … + wn*xn + b

Here: - w represents model weights - x represents input features - b
represents bias or intercept

This linear combination alone is not sufficient for classification,
which leads to the need for a transformation function.

  --------------------------------------------------
  CHAPTER 3: THE SIGMOID FUNCTION
  --------------------------------------------------
  The sigmoid function converts any real-valued
  number into a value between 0 and 1.

  σ(z) = 1 / (1 + e^(-z))

  This property makes it ideal for probability
  estimation. When z is very large, the output
  approaches 1. When z is very small, the output
  approaches 0.
  --------------------------------------------------

CHAPTER 4: PROBABILITY INTERPRETATION

The output of logistic regression is interpreted as a probability.

For example: P(y = 1 | x) = 0.85 means there is an 85% chance the
instance belongs to class 1.

A decision boundary is created using a threshold (commonly 0.5), but
this can be adjusted depending on the problem requirements.

  --------------------------------------------------
  CHAPTER 5: DECISION BOUNDARY
  --------------------------------------------------
  The decision boundary separates classes in the
  feature space. For one feature, it is a point. For
  two features, it is a line. For higher dimensions,
  it becomes a hyperplane.

  Changing the threshold shifts the decision
  boundary and impacts precision and recall.
  --------------------------------------------------

CHAPTER 6: LOSS FUNCTION (LOG LOSS)

Logistic regression uses Binary Cross Entropy loss:

Loss = -[ylog(p) + (1 - y)log(1 - p)]

This loss penalizes confident but incorrect predictions heavily,
encouraging well-calibrated probabilities.

  --------------------------------------------------
  CHAPTER 7: OPTIMIZATION USING GRADIENT DESCENT
  --------------------------------------------------
  The model parameters are learned by minimizing the
  loss function using Gradient Descent.

  Weights are updated iteratively: w = w - α *
  gradient

  Where α is the learning rate. Choosing an
  appropriate learning rate is crucial for
  convergence.
  --------------------------------------------------

CHAPTER 8: REGULARIZATION

Regularization helps control model complexity and prevents overfitting.

L1 Regularization (Lasso) promotes sparsity. L2 Regularization (Ridge)
penalizes large weights.

  --------------------------------------------------
  CHAPTER 9: MULTICLASS LOGISTIC REGRESSION
  --------------------------------------------------
  When there are more than two classes, logistic
  regression is extended using the Softmax function.

  Softmax converts raw scores into a probability
  distribution across all classes.
  --------------------------------------------------

CHAPTER 10: ASSUMPTIONS OF LOGISTIC REGRESSION

-   Linear relationship between features and log-odds
-   Independence of observations
-   No strong multicollinearity
-   Large sample size for stable estimates

  --------------------------------------------------
  CHAPTER 11: EVALUATION METRICS
  --------------------------------------------------
  Common evaluation metrics include: - Accuracy -
  Precision - Recall - F1-score - ROC-AUC

  These metrics help evaluate performance beyond
  simple accuracy.
  --------------------------------------------------

CHAPTER 12: PRACTICAL EXAMPLE

Consider predicting whether a student passes an exam based on study
hours. Logistic regression estimates the probability of passing and
classifies accordingly.

  --------------------------------------------------
  CHAPTER 13: ADVANTAGES AND LIMITATIONS
  --------------------------------------------------
  Advantages: - Simple and interpretable - Efficient
  to train - Probabilistic outputs

  Limitations: - Assumes linear decision boundary -
  Sensitive to outliers - Limited performance on
  complex data
  --------------------------------------------------

CHAPTER 14: REAL-WORLD APPLICATIONS

Logistic regression is widely used in: - Healthcare diagnosis - Fraud
detection - Credit scoring - Marketing analytics

  ---------------------
  CHAPTER 15: SUMMARY
  ---------------------

Logistic Regression is a powerful baseline classification algorithm. Its
simplicity, interpretability, and strong theoretical foundation make it
a core technique in machine learning.
